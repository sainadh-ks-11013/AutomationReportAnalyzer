1. Overview

A Chrome extension that analyzes automation test failure reports on Zoho Corp's internal tools (zohocorpin.com), automatically classifying failures into three categories:

Verdict Indicator Description
Product Issue 🔴 Actual bugs in the product that need developer attention
Automation Issue 🟡 Test infrastructure problems, flaky tests, or expected validation behavior
Needs Review 🔵 Requires manual investigation to determine root cause

3. Core Features

3.1 Failure Pattern Detection

The extension maintains two comprehensive pattern databases for classification:

Automation Issue Patterns (AUTOMATION_PATTERNS)

Category Example Patterns Suggestion
Element Issues NoSuchElementException, StaleElementReferenceException, ElementNotInteractableException, element not found, Absence Of: Check if element locator is correct or add explicit wait
Timing Issues TimeoutException, not available after waiting, Expected condition failed Increase wait time or use dynamic waits
Overlay Issues another element obscures it, alertFreezeLayer, modal Handle popup/overlay before interaction
Setup Issues DUPLICATE_DATA, precondition, associated with other, slot is not found Check test preconditions and data setup
Input Validation cannot be empty, is required, special characters not allowed, please enter Automation provided invalid test data - check input values
Invalid Test Data Validating max characters, invalid domain, invalid key, already exists Test uses intentionally invalid input - expected validation behavior

Product Issue Patterns (PRODUCT_PATTERNS)

Category Severity Example Patterns
Server Errors HIGH 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable
App Exceptions HIGH NullPointerException, ArrayIndexOutOfBoundsException, ClassCastException
API Errors MEDIUM API responded with error, status 40X, status 50X, Request failed
Database Errors HIGH Database error, SQL Exception, Connection refused
Validation Issues MEDIUM expected validation error not displayed, Expected error msg not found
Unexpected Behavior MEDIUM wrongly shown, not redirecting, updating unexpectedly

3.2 Smart Test Data Scenario Detection

The extension intelligently detects when tests are intentionally using invalid input for validation testing:

// Detection patterns include:
/validating.*special characters/i
/enter.*value.*as.*[!@#$%^&*]/i
/check.*validation/i
/max characters.*validation/i
/negative.*test.*scenario/i
/boundary.*test/i
Logic: When a test data validation scenario is detected AND a validation error occurs (like "cannot be empty" or HTTP 400), the failure is classified as Automation Issue because the validation is working correctly - this is expected behavior.

3.6 Deep Analysis (Case History)

The most powerful feature - performs multi-source historical analysis:

Step 1: In This Report Analysis

  • Checks if test ran multiple times in same report
  • Detects "First run passed, second failed" pattern (classic flaky indicator)
  • Counts pass/fail within report runs

Step 2: Same Feature Analysis

  • Scans all rows sharing the same feature name
  • Detects Momentary Issues - when same feature has both passing and failing tests
  • If other tests in same feature pass → likely automation flakiness

Step 3: Historical Analysis (Past Final Results)

  • Extracts test case ID from row (multiple methods: ID attribute, data attributes, ng-click, Angular scope)
  • Calls Aalam Legacy API via background script (bypasses CORS)
  • Analyzes:
    • Total historical executions
    • Pass/Fail counts and failure rate %
    • Consecutive recent failures
    • Pass rate trends
    • Previous classifications (Product vs Automation)

Step 4: Signal Aggregation

Signal Type Weight Example Triggers
Initial panel classification +3 Pattern matched in main analysis
Momentary issue detected +3 (automation) Same feature has passing tests
Mixed pass/fail in report +2 (automation) Flaky test pattern
Consistent failures (>3 consecutive) +2 (product) Recurring issue
High failure rate (>70%) +2 (product) Systemic problem
Previously classified as X +3 Historical classification

4. Data Flow

Page Load ↓ Content Script Injected (waits) ↓ User clicks "Analyze Page" from extension popup ↓ extractAllFailureRows() → DOM parsing ↓ For each row: analyzeFailure() ↓ Pattern matching + test data scenario check ↓ createPanel() → Show floating UI ↓ [User clicks Deep Analyze] ↓ Find Case History menu → Extract links ↓ Background.js → Fetch Aalam API data (CORS bypass) ↓ generateInsight() → Aggregate all signals ↓ openDeepAnalysisReport() → New tab with full analysis

5. Exposed Global API

window.AFI = {
  analyze: init,              // Trigger analysis
  getResults: () => {...},    // Get current results array
  openFullReport: () => {},   // Open full report in new tab
  refresh: init,              // Re-run analysis
  deepAnalyze: (row, idx)     // Deep analyze specific row
};

Usage Examples:

// Trigger analysis programmatically
window.AFI.analyze();

// Get all analyzed results
const results = window.AFI.getResults();
console.log(results.length + " failures found");

// Open full report
window.AFI.openFullReport();

// Deep analyze specific row
window.AFI.deepAnalyze(results[0], 0);

7. Quick Start for Development

  1. Load Extension:
    • Open Chrome → chrome://extensions
    • Enable "Developer mode"
    • Click "Load unpacked"
    • Select the automation-log-analyzer folder
  2. Navigate to Report Page:
    • Open any zohocorpin.com/AutomationReports page
    • Or any URL containing reportsnew or Qap
  3. Run Analysis:
    • Click the extension icon in Chrome toolbar
    • Click "Analyze Page" button
    • Wait 2-3 seconds for analysis to complete
  4. View Results:
    • Floating panel appears with all failures classified
    • Click any row to scroll/highlight original in page
    • Use "Full Report" for exportable tabular view
    • Use "Deep Analyze" for historical comparison
Note: Deep Analysis requires the row to be expanded first (click on it) to reveal the Case History menu.
